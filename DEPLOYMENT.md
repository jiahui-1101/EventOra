# EventOra Deployment Guide

## Environment Variables

Backend:

```text
DB_HOST=
DB_NAME=eventora
DB_USER=
DB_PASS=
JWT_SECRET=
JWT_EXPIRY=86400
FRONTEND_URL=https://your-frontend-domain.example
```

Frontend:

```text
VITE_API_BASE_URL=https://your-backend-domain.example/api
```

## Local Smoke Test

1. Start MySQL and create the schema.
2. Copy `backend/.env.example` to `backend/.env`.
3. Run backend dependencies:

```powershell
cd backend
& "C:\laragon\bin\php\php-8.3.30-Win32-vs16-x64\php.exe" "C:\laragon\bin\composer\composer.phar" install
```

4. Import database scripts:

```powershell
Get-Content -Raw database\schema.sql | & "C:\laragon\bin\mysql\mysql-8.4.3-winx64\bin\mysql.exe" -u root -p
Get-Content -Raw database\create_notifications.sql | & "C:\laragon\bin\mysql\mysql-8.4.3-winx64\bin\mysql.exe" -u root -p
Get-Content -Raw database\seed-complete-demo-data.sql | & "C:\laragon\bin\mysql\mysql-8.4.3-winx64\bin\mysql.exe" -u root -p
```

5. Run backend:

```powershell
& "C:\laragon\bin\php\php-8.3.30-Win32-vs16-x64\php.exe" -S localhost:8000 -t public
```

6. Run frontend:

```powershell
cd ..\frontend
npm.cmd install
npm.cmd run dev
```

## Public Deployment Checklist

- Backend health check returns `{"status":"ok"}`.
- Frontend `VITE_API_BASE_URL` points to the public backend `/api`.
- Backend `FRONTEND_URL` matches the public frontend URL for CORS.
- Database has schema, notifications table, and demo/admin seed data.
- Ticketing smoke flow works: register, mock pay, My Tickets, check-in.
- Android build uses the public API URL, not `localhost`.

## Android Emulator Build

The Android project lives in `frontend/android` and is generated by Capacitor.

Use a public backend URL for Android testing because the emulator cannot use the browser's `localhost:8000` backend directly.

```powershell
cd frontend
$env:VITE_API_BASE_URL="https://your-backend-domain.example/api"
npm.cmd run build
npx.cmd cap sync android
npx.cmd cap open android
```

In Android Studio:

1. Let Gradle sync finish.
2. Start an Android emulator.
3. Run the `app` configuration.
4. Verify login, My Tickets, and organiser manual check-in.

If testing against a local backend instead of public deployment, use `http://10.0.2.2:8000/api` for the Android emulator API URL and ensure the backend CORS `FRONTEND_URL` allows the Android WebView origin used during testing.
